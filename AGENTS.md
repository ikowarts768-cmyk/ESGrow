# ESGrow Project Instructions

## Project Context

Creator: Jonathan Ikowa

Project: ESGrow

Purpose: ESGrow is an ESG tracking system for monitoring Environmental, Social, and Governance performance for companies, with a practical focus on Zambia.

## Creator Background

- Business Administration student
- Graphic designer
- Beginner in Python
- Interested in ESG, investments, and strategy

## Project Goals

- Build a real-world ESG tracking tool
- Keep the project simple and understandable
- Learn Python by building practical features
- Focus on business usefulness, not unnecessary technical complexity

## Tech Preferences

- Use Python as the primary language.
- Prefer a simple project structure.
- Avoid overengineering.
- Write beginner-friendly code.
- Add clear comments where they help explain business logic or Python concepts.

## Current And Future Features

- ESG scoring system
- Company data input
- Basic dashboard or output view
- Possible API integration later
- Deployment, likely using Render
- GitHub version control

## How Codex Should Work On This Project

- Explain planned changes before editing files.
- Do not break working code unnecessarily.
- Debug step by step.
- Keep solutions simple and beginner friendly.
- Suggest improvements, but do not force them.
- Prefer practical business value over advanced technical design.
- When adding code, explain what the code does in plain language.
- When possible, connect technical choices back to ESG, investment, or business strategy use cases.

## Development Style

- Make small, understandable changes.
- Use descriptive variable and function names.
- Keep files focused and easy to read.
- Avoid adding new libraries unless they are clearly useful.
- If a feature can be done with standard Python first, start there.
- Add tests or checks when they help confirm that scoring or calculations work correctly.
